Guide Complet Postman – Tester un Backend Node.js / Express

Refour

Ce document explique comment tester un backend Express avec Postman. Il s’adresse à des débutants et explique tous les concepts fondamentaux.

1. Qu’est-ce que Postman

Postman est un outil qui permet d’envoyer des requêtes HTTP vers un serveur backend sans passer par une interface graphique (frontend).

Postman est utilisé pour :

  • Tester une API pendant le développement
  • Vérifier que les routes fonctionnent
  • Envoyer des données au serveur
  • Lire les réponses retournées par l’API

Postman simule ce que ferait un frontend (React, mobile, etc.).

2. Concepts fondamentaux d’une API

2.1 Le Host

Le host est l’adresse du serveur. 

http://localhost

localhost signifie que le serveur tourne sur votre ordinateur.

2.2 Le Port

Le port identifie l’application sur le serveur. 

http://localhost:3000

Dans notre projet, Express écoute sur le port 3000.

2.3 L’URL complète

Une URL complète est composée de : 

http://localhost:3000/auth/register
  • http : protocole
  • localhost : host
  • 3000 : port
  • /auth/register : endpoint

3. Méthodes HTTP

Les méthodes HTTP indiquent l’action que l’on souhaite effectuer.

  • GET : récupérer des données
  • POST : créer des données
  • PUT : modifier des données
  • DELETE : supprimer des données

Dans ce projet :

  • GET /annonces → récupérer les annonces
  • POST /auth/register → créer un utilisateur
  • POST /annonces/create → créer une annonce

4. Les Headers

Les headers contiennent des informations supplémentaires envoyées au serveur.

Pour une API JSON, le header le plus important est : 

Content-Type: application/json

Cela indique au serveur que le body de la requête est au format JSON.

5. Le Body

Le body contient les données envoyées au serveur.

Dans Postman, on choisit :

  • Body
  • raw
  • JSON

6. Test des endpoints – Cas pratiques

6.1 Créer un utilisateur

Méthode : 

POST

URL : 

http://localhost:3000/auth/register

Headers : 

Content-Type: application/json

Body : 

{
  "email": "test@mail.com",
  "password": "password123"
}

Réponse attendue : 

{
  "message": "Utilisateur créé"
}

6.2 Créer une annonce

Méthode : 

POST

URL : 

http://localhost:3000/annonces/create

Body : 

{
  "titre": "Appartement à louer",
  "description": "Appartement 3 pièces",
  "user_id": 1,
  "category_id": 2
}

Réponse attendue : 

{
  "message": "Annonce créée"
}

6.3 Récupérer toutes les annonces

Méthode : 

GET

URL : 

http://localhost:3000/annonces

Réponse attendue : 

{
  "data": [
    {
      "id": 1,
      "titre": "Appartement à louer",
      "description": "Appartement 3 pièces",
      "user_id": 1,
      "category_id": 2
    }
  ]
}

7. Comprendre les réponses HTTP

  • 200 : requête réussie
  • 201 : ressource créée
  • 400 : erreur de validation
  • 404 : route inexistante
  • 500 : erreur serveur

8. Erreurs courantes

  • Serveur non lancé
  • Mauvais port
  • Mauvais Content-Type
  • Données manquantes dans le body
  • Route mal écrite

Conclusion

Postman est un outil indispensable pour tester et comprendre une API backend. Avant même d’avoir un frontend, il permet de valider toute la logique serveur.

Une fois les tests Postman validés, le frontend peut consommer l’API en toute confiance.
