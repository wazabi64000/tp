Guide Backend Complet – Node.js / Express / MySQL (MVC)

Ce document explique pas à pas comment créer un backend professionnel pour un site de petites annonces. Tout est détaillé : dossiers, fichiers, logique, sécurité, validation, architecture MVC.

1. Création du projet

1.1 Création du dossier

mkdir backend-annonces
cd backend-annonces

1.2 Initialisation Node.js

npm init -y

Cela crée le fichier package.json qui décrit le projet.

1.3 Activation des modules ES

Dans package.json, ajouter : 

{
  "type": "module"
}

Cela permet d’utiliser import et export.

2. Installation des dépendances

npm install express mysql2 dotenv argon2 joi cors
npm install nodemon --save-dev

Rôle des dépendances

  • express : serveur HTTP
  • mysql2 : connexion MySQL
  • dotenv : variables d’environnement
  • argon2 : chiffrement des mots de passe
  • joi : validation des données
  • cors : communication avec le frontend

Fichier .env

PORT=3000
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=
DB_NAME=annonces_db

Ce fichier contient les informations sensibles. Il ne doit jamais être versionné.

5. Connexion à MySQL

config/db.js

import mysql from "mysql2/promise";

const pool = mysql.createPool({
  host: process.env.DB_HOST,
  user: process.env.DB_USER,
  password: process.env.DB_PASSWORD,
  database: process.env.DB_NAME
});

export default pool;

Le pool gère automatiquement les connexions à la base de données.

6. Modèles (Models)

models/user.model.js

import db from "../config/db.js";

export const createUser = async (email, password) => {
  const [result] = await db.query(
    "INSERT INTO users (email, password) VALUES (?, ?)",
    [email, password]
  );
  return result.insertId;
};

export const findUserByEmail = async (email) => {
  const [rows] = await db.query(
    "SELECT * FROM users WHERE email = ?",
    [email]
  );
  return rows[0];
};

models/annonce.model.js

import db from "../config/db.js";

export const getAllAnnonces = async () => {
  const [rows] = await db.query(
    "SELECT * FROM annonces"
  );
  return rows;
};

export const createAnnonce = async (data) => {
  const { titre, description, user_id, category_id } = data;
  await db.query(
    "INSERT INTO annonces (titre, description, user_id, category_id) VALUES (?, ?, ?, ?)",
    [titre, description, user_id, category_id]
  );
};

7. Validation des données

validations/auth.validation.js

import Joi from "joi";

export const registerSchema = Joi.object({
  email: Joi.string().email().required(),
  password: Joi.string().min(6).required()
});

validations/annonce.validation.js

import Joi from "joi";

export const annonceSchema = Joi.object({
  titre: Joi.string().min(3).required(),
  description: Joi.string().required(),
  user_id: Joi.number().required(),
  category_id: Joi.number().required()
});

8. Controllers

controllers/auth.controller.js

import argon2 from "argon2";
import { createUser, findUserByEmail } from "../models/user.model.js";
import { registerSchema } from "../validations/auth.validation.js";

export const register = async (req, res) => {
  const { error } = registerSchema.validate(req.body);
  if (error) {
    return res.status(400).json({ message: error.message });
  }

  const { email, password } = req.body;

  const user = await findUserByEmail(email);
  if (user) {
    return res.status(400).json({ message: "Utilisateur existant" });
  }

  const hashedPassword = await argon2.hash(password);

  await createUser(email, hashedPassword);

  res.status(201).json({ message: "Utilisateur créé" });
};

controllers/annonce.controller.js

import { getAllAnnonces, createAnnonce } from "../models/annonce.model.js";
import { annonceSchema } from "../validations/annonce.validation.js";

export const getAnnonces = async (req, res) => {
  const annonces = await getAllAnnonces();
  res.json({ data: annonces });
};

export const addAnnonce = async (req, res) => {
  const { error } = annonceSchema.validate(req.body);
  if (error) {
    return res.status(400).json({ message: error.message });
  }

  await createAnnonce(req.body);
  res.status(201).json({ message: "Annonce créée" });
};

9. Routes

routes/auth.route.js

import express from "express";
import { register } from "../controllers/auth.controller.js";

const router = express.Router();

router.post("/register", register);

export default router;

routes/annonce.route.js

import express from "express";
import { getAnnonces, addAnnonce } from "../controllers/annonce.controller.js";

const router = express.Router();

router.get("/", getAnnonces);
router.post("/create", addAnnonce);

export default router;

10. Serveur principal

server.js

import express from "express";
import cors from "cors";
import dotenv from "dotenv";

import authRoutes from "./routes/auth.route.js";
import annonceRoutes from "./routes/annonce.route.js";

dotenv.config();

const app = express();

app.use(cors());
app.use(express.json());

app.use("/auth", authRoutes);
app.use("/annonces", annonceRoutes);

app.listen(process.env.PORT, () => {
  console.log("Serveur démarré sur http://localhost:" + process.env.PORT);
});

11. Flux complet d’une requête

  • Le frontend envoie une requête HTTP
  • La route reçoit la requête
  • Le controller valide et traite
  • Le modèle interagit avec MySQL
  • La réponse est renvoyée

Conclusion

Ce backend est une base professionnelle, claire, évolutive et sécurisée. Il est prêt à être connecté à un frontend React.
